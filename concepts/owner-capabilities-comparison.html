Known Limitations (Free / GitHub Pages Tier)

These are the constraints of running on GitHub's free infrastructure. Each one has a known fix if the tool moves to the hosted server.

⏱️ Menu Updates Are Delayed 1–4 Hours
GitHub's free runner queue is shared across all users. Scheduled scrapes are queued and delayed by 1–4 hours past the scheduled time on busy days. If inventory changes in Sweed at 9 AM, the menu may not reflect it until 11 AM or later.
Hosted version runs cron directly on the server — updates fire at the exact scheduled time, no queue.
🚫 No Manual Refresh Button
GitHub automatically revokes any access token embedded in a public HTML page. Multiple attempts were made to add a one-click refresh button — GitHub kills the token every time. The only workaround is going to GitHub → Actions → Run Workflow manually, which requires a GitHub account and takes 5–10 minutes.
Hosted version has a PIN-protected "Refresh Menu Now" button that triggers an immediate scrape in ~30 seconds.
🔬 Mood Scores Are Only as Good as COA Data in Sweed
Every mood score is derived from the terpene data entered into Sweed POS. If terpenes are left blank, the app falls back to general strain knowledge (generic, not batch-specific). If terpenes are copied from a website instead of the actual lab COA, scores reflect brand claims — not the real product. Terpene order matters too: COA position #1 is weighted most heavily.
Operational fix — not a code issue. Staff must enter terpenes from the actual COA PDF in the exact order listed on the lab report, at intake. This is a product intake SOP, not a software limitation.
📅 "New" and "Sold Out" Windows Are Fixed
New in Last 3 Days — if a product was already in Sweed before the scraper first ran, it won't appear as new even if staff have never seen it. Sold Out — Last 2 Days — products gone longer than 2 days disappear from the sold section silently. These windows (3 days / 2 days) are currently hardcoded.
Windows can be changed to any value on request — a 5-minute code change. No architectural issue.
🔓 No Access Control — Public by Default
GitHub Pages is a public URL. Anyone who finds the link can see the full menu. There is no login, no PIN wall for the main menu, and no way to restrict who views the data. Competitor stores could potentially monitor inventory.
Hosted version can add PIN or login protection to the entire page, or specific sections. Already done for the Schedule section in the private version.

What the Hosted Version Unlocks

  • →Instant refresh: Staff clicks a button, menu updates in ~30 seconds. No GitHub account needed, no waiting on a runner queue.
  • →Exact schedule timing: Scrape fires at 7 AM, 11 AM, 3 PM, 6 PM, 8 PM CST on the dot. No queue delay, no GitHub runner congestion.
  • →Staff schedule tab: PIN-protected shift schedule viewer built into the same page — no separate app needed.
  • →Custom domain + SSL: e.g. menu.dinkydope.com or staff.mnlegalcannabis.com — full HTTPS, professional URL.
  • →Access control: Full-page PIN protection if you don't want inventory publicly viewable by competitors.

One Thing That Requires an Operational Change

The mood scores are only as accurate as the terpene data entered into Sweed.

This is not a software limitation — it's a process one. When a new product arrives, staff should open the COA PDF from the brand and enter the terpenes in Sweed in the exact order listed on the lab report (order matters — position #1 is weighted most heavily in the scoring formula).

If terpenes are left blank, the app falls back to general strain knowledge — which is a reasonable approximation but not batch-specific. If they're copied from a brand's website instead of the actual COA, scores reflect marketing claims, not the real product.

Recommendation: Make accurate COA entry a product intake requirement. The mood filter is the differentiator vs. every other cannabis menu tool — it's only as powerful as the data behind it.
